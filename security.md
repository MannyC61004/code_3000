Risk Assessment

##########################################################################################################

This repository primarily contains academic work and assignments, however some potential risks if the data fell into "the wrong hands" include:

Academic Integrity Violations including but not limited to: Plagiarism, Colluding, etc.

Credential Harvesting: If scripts or test files contain hardcoded credentials, attackers could use these to compromise related accounts.


##########################################################################################################

Security Measures

##########################################################################################################

To mitigate these risks, the following steps have been taken:

Credential Management: A .gitignore file is present to prevent sensitive files from being pushed to the remote repository.

Collaborative Review: If shared with peers for group projects, a CODEOWNERS file is implemented to require specific approvals before merging changes into the main branch.
